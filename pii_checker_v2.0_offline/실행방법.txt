[ 실행 방법 ]
  1. pii_checker 폴더 안의 pii_checker.exe 를 더블클릭
     (또는 아래 배치 파일 사용: 실행.bat)
  2. 검은 콘솔창이 열리고 잠시 후 브라우저가 자동으로 열립니다
  3. 브라우저가 열리지 않으면 주소창에 직접 입력:
       http://localhost:5000
  4. 종료: 콘솔창을 닫거나 Ctrl+C

[ v2.0 신규 기능 (v1.0 대비) ]
  ■ 지원 파일 형식 추가
    - Word 문서 (.docx) : 단락/표 내 PII 탐지
    - PDF 파일 (.pdf)   : 페이지 텍스트 추출 후 PII 탐지
    - Excel 파일 (.xlsx/.xls) : 셀 값 + SQL 컬럼 분석
    - XML 파일 (.xml)   : 요소/속성값 PII 탐지
    - ZIP/TAR.GZ 압축   : 내부 파일 자동 해제 후 처리

  ■ 로그 형식 추가
    - PostgreSQL 서버 로그 (standard / pgaudit / CSV)
    - Oracle Audit Trail (멀티라인 블록)
    - MSSQL Audit Log

  ■ PII 탐지 강화
    - 한글 숫자 탐지: 공일공-일이삼사-오육칠팔 → 010-1234-5678
    - 한글 숫자 중간 공백 처리: "공 일 공 ..." → 자동 정규화

[ 점검 가능한 파일 형식 ]
  로그 파일: .log .txt .gz .audit .json .csv
  문서 파일: .docx (Word) .pdf .xlsx/.xls (Excel) .xml
  압축 파일: .zip .tar.gz .tgz (내부 파일 자동 추출)

[ 탐지 항목 ]
  주민등록번호(RRN) · 전화번호 · 신용카드번호 · 계좌번호
  이메일 · 여권번호 · 생년월일 · 이름 · 주소 · 사원번호

[ 샘플 파일 위치 ]
  sample_logs/   - DB/웹/앱 로그 샘플 (MySQL, PostgreSQL, Oracle, MSSQL 등)
  sample_docs/   - 문서 샘플 (Word, Excel, XML)
  날짜 범위: 2024-02-01 ~ 2024-03-31

[ 주의 사항 ]
  - 동시에 하나의 인스턴스만 실행 가능 (포트 5000)
  - 5000번 포트가 사용 중이면 다른 프로그램 종료 후 재시작
  - 보고서는 실행파일 옆 reports/ 폴더에 저장됩니다
  - 분석 이력은 history/ 폴더에 저장됩니다
