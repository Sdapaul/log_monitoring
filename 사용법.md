## 1. 설치 및 실행

### 방법 A — 오프라인 패키지 (내부망·폐쇄망 권장)

Python 설치 없이 바로 실행합니다.

1. `pii_checker_v4.0_offline.zip` 압축 해제
2. `실행.bat` 더블클릭
3. 콘솔 창에 `Running on http://0.0.0.0:5000` 메시지 확인
4. 브라우저에서 `http://localhost:5000` 접속

```
pii_checker_v4.0_offline/
├── 실행.bat          ← 이 파일 더블클릭
├── 안내.txt
├── app/              ← 프로그램 소스 (수정 가능)
├── python/           ← Python 3.13 런타임 (삭제 금지)
├── lib/              ← 필요 라이브러리 (삭제 금지)
├── sample_logs/      ← 테스트용 샘플 로그
└── sample_docs/      ← 테스트용 샘플 문서
```

> `python/` 및 `lib/` 폴더는 절대 삭제하지 마세요.

---

### 방법 B — Python 직접 실행 (개발/소스 환경)

**사전 요구사항:** Python 3.10 이상

```bash
# 패키지 설치 (최초 1회)
pip install flask openpyxl python-docx pdfplumber

# 서버 실행
python web_app.py
```

브라우저에서 `http://localhost:5000` 접속

**포트 변경:**
```bash
python web_app.py --port 8080
```

## 2. 웹 UI 사용법

### 2-1. 분석 기간 설정

| 버튼 | 설명 |
|---|---|
| 최근 7일 | 오늘 기준 7일 전 ~ 오늘 |

날짜 직접 입력 형식: `YYYY-MM-DD`

---

### 2-2. 파일 등록

**방법 1 — 파일 업로드**

업로드 영역에 드래그앤드롭하거나 클릭하여 선택합니다. 여러 파일 동시 선택 가능 (파일당 최대 500 MB).

| 분류 | 지원 확장자 |
|---|---|
| 로그 파일 | `.log` `.txt` `.gz` `.audit` `.json` `.csv` |
| 문서 파일 | `.docx` `.pdf` `.xlsx` `.xls` `.xlsm` `.pptx` `.ppt` `.xml` |
| 압축 파일 | `.zip` `.tar.gz` `.tgz` — 내부 파일 **자동 해제** 후 분석 |

**방법 2 — 서버 경로 직접 입력**

서버에 이미 있는 파일 경로를 직접 입력합니다. glob 패턴 및 여러 경로(줄 바꿈 구분) 사용 가능.

```
예시:
C:\logs\db_audit_2024.log
C:\logs\web_access\*.log
C:\reports\개인정보취급대장.xlsx
```

---

### 2-3. 점검 항목 선택

| 항목 | 설명 |
|---|---|
| **오남용 탐지** | 개인정보 노출, 업무 외 시간 조회 탐지 |
| **과다조회 탐지** | 일별/시간별 조회 과다, 대량 내보내기, 추세 급증 탐지 |

둘 다 선택하면 전체 분석, 하나만 선택하면 해당 항목만 집계합니다.

## 4. 보고서 다운로드

결과 화면 상단의 **Excel 다운로드** / **HTML 다운로드** 버튼 클릭

### Excel 보고서 — 7개 시트 구성

| 시트 | 내용 |
|---|---|
| 1. 요약 | 분석 기간, 통계, PII 유형별 건수 |
| 2. 사용자별 현황 | 전체 사용자 위험 현황 테이블 |
| 3. 개인정보 검출 상세 | 탐지된 PII 유형·심각도·**검출원본값** |
| 4. SQL 분석 상세 | SELECT절 PII 필드, SELECT* 여부, 노출유형 |
| 5. 소명 요청 양식 | 소명 요청 순위, 긴급도, 위반사유, 질문 |
| 6. 소명 증거 상세 | 핵심 증적·**검출원본값**·화면 추정 노출 |
| 7. 비교 분석 | 1주/1개월 전 대비 증감 현황 |

> **검출원본값**: 실제 탐지된 PII 원본 값이 시트3·시트6에 표시됩니다 (마스킹 없음). 보고서 보안에 유의하세요.

### HTML 보고서

- 인터넷 연결 없이 열리는 단일 파일 (CSS/JS 내장)
- 브라우저에서 바로 인쇄 또는 PDF 저장 가능

## 5. 소명 요청 기능

CRITICAL/HIGH 등급 사용자 중 즉각 확인이 필요한 순서를 자동 산정합니다.

### 긴급도 구분

| 긴급도 | 점수 기준 | 의미 |
|---|---|---|
| 즉시 | 100점 이상 | 당일 소명 요청 |
| 긴급 | 60점 이상 | 금주 내 소명 요청 |
| 검토 | 60점 미만 | 정기 검토 |

### 소명 카드 정보

각 카드를 클릭하면 상세 모달이 열립니다:

- **위반 사유:** 구체적 위반 내용 (한국어 서술)
- **화면 추정 노출:** 실제 화면에 표시됐을 개인정보 예상 내용
- **핵심 증적:** 실제 SQL/접속 기록 상위 5건 (검출원본값 포함)
- **소명 질문:** 당사자에게 물어볼 질문 목록 (복사 가능)
- **이메일 제목:** 소명 요청 이메일 제목으로 바로 사용 가능

### 소명 점수 산정 기준

| 항목 | 최대 점수 |
|---|---|
| 단일 쿼리 대량 노출 (20,000건 이상) | 55점 |
| 기간 내 총 노출량 (10,000건 이상) | 40점 |
| 야간 개인정보 조회 (20건 이상) | 40점 |
| 일별 과다조회 | 30점 |
| 시간별 과다조회 | 20점 |
| 대량 조회 (BULK) | 20점 |
| PII 유형 다양성 | 15점 |
| 최고위험 등급 배수 | ×1.25 |

## 6. CLI 사용법 (고급)

웹 UI 없이 명령줄에서 직접 실행합니다.

```bash
python main.py \
  --log-files C:\logs\db_audit.log C:\logs\web_access.log \
  --start-date 2024-03-01 \
  --end-date 2024-03-31 \
  --output-dir C:\reports\march
```

### 주요 옵션

| 옵션 | 기본값 | 설명 |
|---|---|---|
| `--log-files` | (필수) | 로그/문서 파일 경로, glob 패턴 가능 |
| `--start-date` | (필수) | 분석 시작일 (YYYY-MM-DD) |
| `--end-date` | (필수) | 분석 종료일 (YYYY-MM-DD) |
| `--output-dir` | `./reports` | 보고서 저장 경로 |
| `--format` | `auto` | 로그 형식 (`auto`/`db`/`postgresql`/`web`/`app`/`generic`) |
| `--report-format` | `both` | 보고서 형식 (`html`/`excel`/`both`) |
| `--min-risk` | `LOW` | 최소 위험 등급 |
| `--check-misuse` | 켜짐 | 오남용 탐지 여부 |
| `--check-excess` | 켜짐 | 과다조회 탐지 여부 |
| `--threshold-file` | 없음 | 임계값 오버라이드 JSON 경로 |

### 사용 예시

```bash
# DB 로그, Excel 보고서만
python main.py --log-files db.log --start-date 2024-01-01 --end-date 2024-01-31 \
  --format db --report-format excel

# PostgreSQL 로그
python main.py --log-files pg_log.log --start-date 2024-01-01 --end-date 2024-01-31 \
  --format postgresql

# Word/Excel/PPTX 문서 점검
python main.py --log-files 개인정보취급대장.docx 접근로그.xlsx 발표자료.pptx \
  --start-date 2024-01-01 --end-date 2024-12-31

# HIGH 이상만, 오남용 탐지만
python main.py --log-files *.log --start-date 2024-01-01 --end-date 2024-01-31 \
  --min-risk HIGH --check-excess false
```

## 7. 임계값 설정 변경

`config.py`를 직접 수정하거나 JSON 파일로 오버라이드합니다.

### 주요 임계값 (config.py → THRESHOLDS)

**오남용 관련**

| 항목 | 기본값 | 설명 |
|---|---|---|
| `business_hours_start` | 8 | 업무 시작 시간 (24h) |
| `business_hours_end` | 19 | 업무 종료 시간 (24h) |
| `after_hours_threshold` | 5 | 업무 외 시간 PII 조회 경고 기준 |
| `pii_single_query_high` | 5,000 | 단일 쿼리 노출량 고위험 기준 |
| `pii_single_query_critical` | 20,000 | 단일 쿼리 노출량 최고위험 기준 |

**과다조회 관련**

| 항목 | 기본값 | 설명 |
|---|---|---|
| `max_queries_per_day` | 500 | 일별 쿼리 수 경고 기준 |
| `critical_queries_per_day` | 2,000 | 일별 쿼리 수 위험 기준 |
| `max_queries_per_hour` | 100 | 시간당 쿼리 수 경고 기준 (슬라이딩 1h) |
| `critical_queries_per_hour` | 500 | 시간당 쿼리 수 위험 기준 |
| `bulk_select_row_threshold` | 1,000 | 대량 조회 판정 행 수 기준 |
| `bulk_export_per_day` | 3 | 하루 대량 조회 경고 기준 |
| `unique_targets_per_day` | 50 | 하루 고유 대상 접근 수 경고 기준 |
| `daily_surge_threshold_pct` | 10 | 일별 추세 급증 탐지 기준 (%) |
| `daily_surge_min_baseline_days` | 7 | 추세 탐지 최소 과거 일수 |

### JSON 파일로 오버라이드

```json
{
  "max_queries_per_day": 300,
  "business_hours_start": 9,
  "business_hours_end": 18,
  "after_hours_threshold": 3
}
```

CLI 실행 시 적용:
```bash
python main.py --log-files audit.log ... --threshold-file my_thresholds.json
```

## 8. 지원 파일 형식

### 로그 파일 — 자동감지 지원 형식

| 형식 | 파서 | 예시 패턴 |
|---|---|---|
| MySQL General Log | DbAccessParser | `2024-01-15T10:23:45Z 42 Query SELECT ...` |
| Oracle Audit Trail | DbAccessParser | `DB USER: SCOTT ... SQL TEXT: SELECT ...` |
| MSSQL Audit Log | DbAccessParser | `LoginName=hong;StatementText=SELECT...;RowsAffected=100` |
| PostgreSQL Server Log | DbAccessParser | `[1234] user@db LOG: statement: SELECT ...` |
| PostgreSQL pgaudit | DbAccessParser | `LOG: AUDIT: SESSION,1,1,READ,SELECT,...` |
| PostgreSQL CSV Log | DbAccessParser | `2024-01-15 10:23:45,user,db,...,SELECT...` |
| Apache/Nginx Combined | WebAccessParser | `192.168.1.1 - - [15/Jan/2024] "GET /api/..."` |
| Log4j / Python logging | AppLogParser | `2024-01-15 10:23:45,123 INFO user=hong ...` |
| Syslog RFC3164/5424 | AppLogParser | `Jan 15 10:23:45 server app: SELECT ...` |
| 비정형 텍스트 | GenericParser | 구조 없는 텍스트, 숫자 패턴 직접 탐지 |

### 문서 파일

| 형식 | 추출 엔진 | 추출 내용 |
|---|---|---|
| Word (.docx) | python-docx | 단락 텍스트 + 표 셀 |
| PDF (.pdf) | pdfplumber | 페이지별 텍스트 (스캔형 미지원) |
| Excel (.xlsx / .xls / .xlsm) | openpyxl | 시트별 셀 값 |
| **PowerPoint (.pptx / .ppt)** | zipfile+XML (내장) | 슬라이드 텍스트 (python-pptx 불필요) |
| XML (.xml) | ElementTree | 요소 텍스트 + 속성값 |

### 압축 파일

| 형식 | 처리 방식 |
|---|---|
| `.zip` | 내부 로그/문서 파일 자동 추출 |

## 9. 탐지 항목 설명

### 오남용 탐지

| 항목 | 탐지 기준 | 위험도 |
|---|---|---|
| **개인정보 노출 (PII_EXPOSURE)** | 로그/문서에서 PII 패턴 직접 탐지, 또는 SQL SELECT절 PII 컬럼 포함 | 최고위험~중위험 |
| **단일 대량 노출 (PII_RECORD_EXPOSURE)** | 실효 노출량(결과건수×PII컬럼수) ≥5,000 고위험 / ≥20,000 최고위험 | 최고위험~고위험 |
| **업무외시간 조회 (AFTER_HOURS)** | 08:00~19:00 외 시간 PII 조회 5건 초과 | 고위험 |

### 과다조회 탐지

| 항목 | 탐지 기준 | 위험도 |
|---|---|---|
| **일별 과다조회 (EXCESSIVE_ACCESS)** | 일 ≥500 고위험 / ≥2,000 최고위험 | 최고위험~고위험 |
| **시간별 과다조회 (EXCESSIVE_ACCESS)** | 1시간 내 ≥100 고위험 / ≥500 최고위험 | 최고위험~고위험 |
| **대량 내보내기 (BULK_EXPORT)** | 결과 1,000건 이상 조회, 하루 3건 초과 | 고위험 |
| **고유 대상 다양성 (EXCESSIVE_ACCESS)** | 하루 50명(건) 초과 개별 대상 조회 | 중위험 |
| **일별 추세 급증 (EXCESSIVE_ACCESS)** | 최근 30일 평균 대비 10% 초과 (50% 초과 시 고위험) | 고위험~중위험 |

> 추세 급증 탐지는 최소 7일 이상의 이력이 쌓인 후 동작합니다.

| 주민등록번호 | RRN | 최고위험 | `800101-1000008` (체크섬+성별코드 1~9 검증) |
| 계좌번호 | ACCOUNT_NO | 최고위험 | `계좌: 1234567890123` |
| 신용카드번호 | CREDIT_CARD | 최고위험 | `4111-1111-1111-1111` (Luhn 검증) |
| 이름 (쿼리 내) | NAME_IN_QUERY | 고위험 | `cust_name='홍길동'` |
| 전화번호 | PHONE | 고위험 | `010-1234-5678` |
| 생년월일 | BIRTHDATE | 고위험 | `birth=1990-01-15` |
| 여권번호 | PASSPORT | 고위험 | `M12345678` |
| 이메일 | EMAIL | 중위험 | `hong@example.com` |
| 주소 | ADDRESS | 중위험 | `서울시 강남구 테헤란로 123` |

> **신용카드**: Luhn 알고리즘 검증  
> **사원번호(독립형)**: 앞에 다른 숫자가 없는 경우만 탐지 (예: 타임스탬프 오탐 방지)

### 노출 유형 구분

| 노출유형 | 판단 기준 | 위험도 |
|---|---|---|
| **전체컬럼노출** | `SELECT *` 또는 `SELECT t.*` 사용 | 높음 (PII 컬럼 수 3~5개 가정) |
| **PII컬럼직접출력** | SELECT절에 PII 컬럼명 직접 명시 | 중간 (실제 컬럼 수 반영) |
| **검색조건만사용** | WHERE절에만 PII 사용, SELECT에는 없음 | 낮음 (결과에 PII 미포함) |

---

### 지표 상세 해설: PII쿼리수 · PII컬럼직접출력 · 검색조건만사용

**PII쿼리수** — "실제 PII 값이 로그에 있었던 쿼리 수"

```python
# log_event.py
def has_pii(self) -> bool:
    return len(self.pii_hits) > 0   # pii_hits = 정규식으로 탐지된 실제 값
```

`pii_detector`가 로그 원문 전체(쿼리 텍스트, 파라미터, 결과 등)에서 실제 주민번호·전화번호·이메일 등 값 패턴을 찾아낸 이벤트 수. SQL 구조와 무관합니다.

**PII컬럼직접출력** — "SELECT에 PII 컬럼명을 명시한 쿼리"

```python
# sql_clause_analyzer.py
result.exposure_type = 'FULL_EXPOSURE' if result.is_select_star else 'PARTIAL_EXPOSURE'
```

`SELECT name, phone, rrn FROM ...` 처럼 컬럼명을 구체적으로 썼고 그 중에 PII 키워드가 있는 경우. 실제 값이 로그에 없어도 컬럼명만으로 분류됩니다.

**검색조건만사용** — "WHERE에만 PII 컬럼을 쓴 쿼리"

```python
# sql_clause_analyzer.py
elif result.where_pii_fields:
    result.exposure_type = 'SEARCH_ONLY'
```

`SELECT order_id FROM orders WHERE jumin='...'` 처럼 결과에는 PII가 없고 검색 조건으로만 사용한 경우.

---

### 세 지표 비교 예시

**예시 1)** `SELECT name, phone FROM members WHERE jumin='720101-1234567'`

| 지표 | 해당? | 이유 |
|---|---|---|
| PII쿼리수 | O | jumin 값(RRN 패턴)이 로그 원문에 존재 |
| PII컬럼직접출력 | O | SELECT에 name, phone 명시 |
| 검색조건만사용 | X | SELECT에 이미 PII 있으므로 PARTIAL_EXPOSURE로 분류 |

**예시 2)** `SELECT order_id FROM orders WHERE jumin='720101-1234567'`

| 지표 | 해당? | 이유 |
|---|---|---|
| PII쿼리수 | O | jumin 값이 로그에 노출 |
| PII컬럼직접출력 | X | SELECT에 PII 컬럼 없음 |
| 검색조건만사용 | O | WHERE에만 PII |

**예시 3)** `SELECT name, phone FROM members WHERE id=1` *(결과값은 로그에 없음)*

| 지표 | 해당? | 이유 |
|---|---|---|
| PII쿼리수 | X | 로그 원문에 실제 PII 값 패턴 없음 |
| PII컬럼직접출력 | O | SELECT에 name, phone 명시 |
| 검색조건만사용 | — | SELECT에 PII가 있으므로 PARTIAL_EXPOSURE, 검색조건만사용 미집계 |

> **핵심 차이:** PII쿼리수는 "데이터 값이 실제로 로그에 찍혔는가", PII컬럼직접출력·검색조건만사용은 "SQL 구조상 어떤 의도로 PII를 사용했는가"를 봅니다.

## 12. 자주 묻는 질문

**Q. 브라우저가 자동으로 열리지 않아요.**  
→ 콘솔에 `Running on http://0.0.0.0:5000` 메시지가 나오면 브라우저에서 `http://localhost:5000` 직접 입력.

**Q. 분석 결과에 사용자가 0명이에요.**  
→ 분석 기간과 로그 파일의 날짜가 일치하는지 확인하세요. 로그 형식을 `자동감지` → 다른 형식으로 바꿔 보세요.

**Q. PDF를 올렸는데 탐지가 안 돼요.**  
→ 스캔된 이미지형 PDF는 텍스트 추출이 불가합니다. 텍스트 레이어가 있는 PDF만 지원합니다.

**Q. PPTX를 올렸는데 오류가 나요.**  
→ `python-pptx` 미설치 환경에서도 동작합니다. 오프라인 패키지(v4.0_offline.zip)를 사용하세요.

**Q. Excel 파일 업로드 시 "압축 파일 오류"가 나요.**  
→ xlsx/xlsm은 ZIP 구조이므로 정상 Excel 파일이 맞는지 확인하세요. 손상된 파일은 재저장 후 시도하세요.

**Q. 이전 대비 비교가 표시되지 않아요.**  
→ 최소 2번 이상 분석을 실행해야 이력이 쌓입니다. 처음 실행 시에는 비교 데이터가 없습니다.

**Q. 포트 5000이 이미 사용 중이에요.**  
→ `python web_app.py --port 8080` 으로 변경하세요.

**Q. 다른 PC에서 접속하고 싶어요.**  
→ 서버 PC의 IP를 확인 후 `http://[서버IP]:5000` 으로 접속. 방화벽에서 해당 포트 허용 필요.

**Q. 분석 중 오류가 발생해요.**  
→ 웹 UI 진행 로그 영역 하단에 오류 내용이 표시됩니다. `/api/status/<job_id>` 응답의 `error` 필드도 확인하세요.

**Q. 코드를 수정했는데 반영이 안 돼요.**  
→ `python web_app.py` 를 재시작해야 합니다. 실행 중인 서버는 코드 변경을 자동 반영하지 않습니다.

### 처리 방식: 단일 머신 완결형

- **데이터베이스 없음**: 분석 결과는 `history/*.json`, 보고서는 `reports/` 폴더에만 저장
- **외부 서버 통신 없음**: 인터넷, 클라우드, 외부 API 호출 전무
- **메모리 기반 작업 관리**: 분석 Job은 메모리 딕셔너리(`JOBS`)에서 관리, 서버 재시작 시 초기화

### 인증 및 접근 제어 — 내부망 운용 지침

> 본 프로그램은 **단일 보안 담당자 워크스테이션에서 localhost로 운용**하는 것을 전제로 설계되었습니다.

| 항목 | 현재 상태 | 권장 조치 |
|---|---|---|
| 웹 UI 로그인 | **없음** | localhost(127.0.0.1) 바인딩 유지. 타 PC 접근 필요 시 OS 방화벽으로 포트 5000 제한 |
| HTTPS | **미적용** | 단일 PC 로컬 운용이면 문제 없음. 네트워크 공유 시 SSH 터널 권장 |
| 보고서 파일 접근 | 파일 시스템 권한으로만 보호 | `reports/` 폴더를 보안 담당자 계정만 접근 가능하도록 ACL 설정 |
| 이력 데이터 접근 | 파일 시스템 권한으로만 보호 | `history/` 폴더 동일하게 ACL 적용 |

**localhost 전용 실행 (권장):**
```bash
python web_app.py --host 127.0.0.1 --port 5000
```

**기본 실행 시 바인딩:**
```
Running on http://0.0.0.0:5000  ← 모든 네트워크 인터페이스
```
→ 내부망에서도 같은 PC의 다른 계정이나 동일 서브넷 PC에서 접근 가능하므로, 단독 운용 시 `--host 127.0.0.1` 옵션을 사용하거나 OS 방화벽 설정을 권장합니다.
