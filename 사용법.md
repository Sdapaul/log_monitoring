# 개인정보 오남용·과다조회 점검 시스템 사용법 (v2.0)

> DB·웹·앱 로그는 물론 Word·PDF·Excel·XML 문서까지 분석하여

## 1. 설치 방법

### 방법 A — EXE 파일 (폐쇄망 권장)

별도 설치 없이 바로 실행 가능합니다.

```
pii_checker/
├── pii_checker.exe   ← 이 파일 실행
└── _internal/        ← 삭제하지 마세요 (Python 런타임 내장)
```

1. `pii_checker_v2.0_offline.zip` 압축 해제
2. `실행.bat` 더블클릭 (또는 `pii_checker.exe` 직접 실행)
3. 브라우저가 자동으로 열립니다 → `http://localhost:5000`

> **주의:** `_internal` 폴더는 exe와 같은 위치에 있어야 합니다. 절대 삭제하지 마세요.

---

### 방법 B — Python 직접 실행

**사전 요구사항:** Python 3.10 이상

```bash
# 패키지 설치 (최초 1회)
pip install flask openpyxl python-docx pdfplumber

# 서버 실행
cd C:\test_log_monitoring
python web_app.py
```

브라우저에서 `http://localhost:5000` 접속

**포트 변경이 필요한 경우:**
```bash
python web_app.py --port 8080
```

## 3. 웹 UI 사용법

### 3-1. 분석 기간 설정

| 버튼 | 설명 |
|---|---|
| 최근 7일 | 오늘 기준 7일 전 ~ 오늘 |
| 최근 30일 | 오늘 기준 30일 전 ~ 오늘 |
| 최근 90일 | 오늘 기준 90일 전 ~ 오늘 |
| 이번 달 | 이번 달 1일 ~ 오늘 |

날짜를 직접 입력할 수도 있습니다 (형식: `YYYY-MM-DD`)

---

### 3-2. 파일 등록

**방법 1 — 파일 업로드**
- 파일을 업로드 영역으로 드래그앤드롭하거나 클릭하여 선택
- 여러 파일 동시 선택 가능 / 파일당 최대 500 MB
- 지원 확장자:

| 분류 | 확장자 |
|---|---|
| 로그 파일 | `.log` `.txt` `.gz` `.audit` `.json` `.csv` |
| 문서 파일 | `.docx` (Word) `.pdf` `.xlsx` `.xls` (Excel) `.xml` |
| 압축 파일 | `.zip` `.tar.gz` `.tgz` — 내부 파일 **자동 해제** 후 분석 |

> **문서 파일 처리 방식:**  
> - Word: 단락 + 표 셀 텍스트 추출  
> - PDF: 페이지별 텍스트 추출 (스캔 이미지형 PDF는 미지원)  
> - Excel: 시트별 행 → 탭 구분 텍스트 변환  
> - XML: 요소 텍스트 + 속성값 구조화 추출

**방법 2 — 서버 경로 직접 입력**
- 서버에 이미 있는 파일 경로를 직접 입력
- glob 패턴 사용 가능: `C:\logs\db\*.log`
- 여러 경로는 줄 바꿈으로 구분

```
예시:
C:\logs\db_audit_2024.log
C:\logs\web_access\*.log
C:\reports\개인정보취급대장.xlsx
```

### 3-3. 점검 항목 선택

| 항목 | 설명 |
|---|---|
| **오남용 탐지** | 개인정보 노출, 업무외 시간 조회 탐지 |
| **과다조회 탐지** | 비정상적으로 많은 조회 횟수 탐지 |

둘 다 선택하면 전체 분석, 하나만 선택하면 해당 항목만 집계합니다.

---

### 3-4. 분석 옵션

| 옵션 | 기본값 | 설명 |
|---|---|---|
| 최소 위험 등급 | LOW | LOW 이상만 결과에 표시 |
| 로그 형식 | 자동감지 | 형식을 직접 지정할 수 있음 |
| 보고서 형식 | Excel + HTML | 원하는 형식 선택 |

**로그 형식 선택 가이드:**

| 선택값 | 적용 대상 |
|---|---|
| 자동감지 (기본) | 대부분의 경우 자동 식별 |
| DB 감사 로그 | MySQL / Oracle / MSSQL |
| PostgreSQL | PostgreSQL 표준 / pgaudit / CSV 형식 |
| 웹 액세스 로그 | Apache / Nginx |
| 앱 로그 | Log4j / Python logging / Syslog |
| 범용(비정형) | 구조 없는 텍스트, .txt, 문서 파일 |

## 4. 분석 결과 보기

| 카드 | 설명 |
|---|---|
| 분석 대상자 | 로그에서 발견된 전체 사용자 수 |
| 위험/고위험 | CRITICAL + HIGH 등급 사용자 수 |
| PII 탐지 건수 | 개인정보 패턴이 발견된 총 이벤트 수 |
| 총 분석 이벤트 | 처리된 전체 로그 라인 수 |

---

### 위험 등급 기준

| 등급 | 색상 | 의미 |
|---|---|---|
| **CRITICAL (위험)** | 빨강 | 즉각 조사 필요 |
| **HIGH (고위험)** | 주황 | 우선 검토 필요 |
| **MEDIUM (중위험)** | 파랑 | 모니터링 필요 |
| **LOW (저위험)** | 초록 | 참고 수준 |

---

### 사용자별 위험 테이블 주요 컬럼

| 컬럼 | 설명 |
|---|---|
| 사용자ID | 로그에서 추출된 계정 |
| 위험등급 | CRITICAL / HIGH / MEDIUM / LOW |
| 위험점수 | 0~100점 (높을수록 위험) |
| PII 노출 레코드 | 개인정보가 포함된 조회 결과 건수 |
| 단일쿼리 최대 노출 | 1회 쿼리에서 최대 노출된 개인정보 건수 |
| 총 쿼리 수 | 분석 기간 내 전체 쿼리 수 |
| 야간 조회 | 업무 외 시간(19시~08시) 개인정보 조회 수 |
| 이전 대비 | 지난주/지난달 대비 증감 (▲↑ 증가, ▼↓ 감소) |

결과 화면 상단의 **Excel 다운로드** / **HTML 다운로드** 버튼 클릭

| 시트 | 내용 |

## 6. 소명 요청 기능

CRITICAL/HIGH 등급 사용자 중 **즉각 확인이 필요한 순서**를 자동 산정합니다.

### 긴급도 색상 구분

| 색상 | 긴급도 | 점수 기준 |
|---|---|---|
| 빨강 | 즉시 | 100점 이상 |
| 주황 | 긴급 | 60점 이상 |
| 노랑 | 검토 | 60점 미만 |

### 소명 카드 정보

각 카드를 클릭하면 상세 모달이 열립니다:

- **위반 사유:** 구체적 위반 내용 (한국어 서술)
- **화면 추정 노출:** 실제 화면에 표시됐을 개인정보 예상 내용
- **핵심 증적:** 마스킹 처리된 실제 SQL/접속 기록 상위 5건
- **소명 질문:** 당사자에게 물어볼 질문 목록 (복사해서 사용 가능)
- **이메일 제목:** 소명 요청 이메일 제목으로 바로 사용 가능한 한 줄 요약

### 소명 점수 산정 기준

| 항목 | 최대 점수 |
|---|---|
| 단일 쿼리 대량 노출 (20,000건 이상) | 55점 |
| 기간 내 총 노출량 (10,000건 이상) | 40점 |
| 야간 개인정보 조회 (20건 이상) | 40점 |
| 일별 과다조회 | 30점 |
| 시간별 과다조회 | 20점 |
| 대량 조회(BULK) | 20점 |
| PII 유형 다양성 | 15점 |
| CRITICAL 등급 배수 | ×1.25 |

## 7. CLI 사용법 (고급)

웹 UI 없이 명령줄에서 직접 실행합니다.

```bash
python main.py \
  --log-files C:\logs\db_audit.log C:\logs\web_access.log \
  --start-date 2024-03-01 \
  --end-date 2024-03-31 \
  --output-dir C:\reports\march
```

### 주요 옵션

| 옵션 | 기본값 | 설명 |
|---|---|---|
| `--log-files` | (필수) | 로그/문서 파일 경로, 여러 개 또는 glob 패턴 |
| `--start-date` | (필수) | 분석 시작일 (YYYY-MM-DD) |
| `--end-date` | (필수) | 분석 종료일 (YYYY-MM-DD) |
| `--output-dir` | `./reports` | 보고서 저장 경로 |
| `--format` | `auto` | 로그 형식 (`auto`/`db`/`postgresql`/`web`/`app`/`generic`) |
| `--report-format` | `both` | 보고서 형식 (`html`/`excel`/`both`) |
| `--min-risk` | `LOW` | 최소 위험 등급 |
| `--check-misuse` | 켜짐 | 오남용 탐지 여부 |
| `--check-excess` | 켜짐 | 과다조회 탐지 여부 |
| `--threshold-file` | 없음 | 임계값 오버라이드 JSON 경로 |

### 사용 예시

```bash
# DB 로그만, Excel 보고서만
python main.py --log-files db.log --start-date 2024-01-01 --end-date 2024-01-31 \
  --format db --report-format excel

# PostgreSQL 로그
python main.py --log-files pg_log.log --start-date 2024-01-01 --end-date 2024-01-31 \
  --format postgresql

# Word/Excel 문서 점검
python main.py --log-files 개인정보취급대장.docx 접근로그.xlsx \
  --start-date 2024-01-01 --end-date 2024-12-31

# HIGH 이상만, 오남용 탐지만
python main.py --log-files *.log --start-date 2024-01-01 --end-date 2024-01-31 \
  --min-risk HIGH --check-excess false
```

## 8. 임계값 설정 변경

`config.py`를 직접 수정하거나, JSON 파일로 오버라이드할 수 있습니다.

### 주요 임계값 (config.py)

| 항목 | 기본값 | 설명 |
|---|---|---|
| `max_queries_per_day` | 500 | 일별 쿼리 수 경고 기준 |
| `critical_queries_per_day` | 2,000 | 일별 쿼리 수 위험 기준 |
| `max_queries_per_hour` | 100 | 시간당 쿼리 수 경고 기준 |
| `business_hours_start` | 8 | 업무 시작 시간 (24h) |
| `business_hours_end` | 19 | 업무 종료 시간 (24h) |
| `after_hours_threshold` | 5 | 업무 외 시간 PII 조회 경고 기준 |
| `bulk_select_row_threshold` | 1,000 | 대량 조회 행 수 기준 |
| `pii_exposure_critical` | 50,000 | PII 총 노출량 위험 기준 |
| `pii_single_query_critical` | 20,000 | 단일 쿼리 노출량 위험 기준 |

### JSON 파일로 오버라이드

`my_thresholds.json` 파일 생성:

```json
{
  "max_queries_per_day": 300,
  "business_hours_start": 9,
  "business_hours_end": 18,
  "after_hours_threshold": 3
}
```

CLI 실행 시 적용:
```bash
python main.py --log-files audit.log ... --threshold-file my_thresholds.json
```

## 9. 지원 파일 형식

### 로그 파일

`자동감지(auto)` 모드에서 아래 형식을 자동으로 식별합니다.

| 형식 | 파서 | 예시 |
|---|---|---|
| MySQL General Log | DbAccessParser | `2024-01-15T10:23:45Z 42 Query SELECT ...` |
| **PostgreSQL Server Log** | DbAccessParser | `2024-01-15 10:23:45 UTC [1234] user@db LOG: statement: SELECT ...` |
| **PostgreSQL pgaudit** | DbAccessParser | `LOG: AUDIT: SESSION,1,1,READ,SELECT,...` |
| **PostgreSQL CSV Log** | DbAccessParser | `2024-01-15 10:23:45,user,db,...,SELECT...` |
| Oracle Audit Trail | DbAccessParser | `DB USER: SCOTT ... SQL TEXT: SELECT ...` |
| MSSQL Audit Log | DbAccessParser | `LoginName=hong;StatementText=SELECT...;RowsAffected=100` |
| Apache/Nginx Combined | WebAccessParser | `192.168.1.1 - - [15/Jan/2024:10:23:45] "GET /api/..." 200` |
| Log4j / Python logging | AppLogParser | `2024-01-15 10:23:45,123 INFO user=hong ...` |
| Syslog RFC3164/5424 | AppLogParser | `Jan 15 10:23:45 server app: SELECT ...` |
| 비정형 텍스트 | GenericParser | 구조 없는 텍스트, 숫자 패턴 직접 탐지 |

### 문서 파일 (v2.0 신규)

| 형식 | 추출 엔진 | 추출 내용 |
|---|---|---|
| Word (.docx) | python-docx | 단락 텍스트 + 표 셀 |
| PDF (.pdf) | pdfplumber | 페이지별 텍스트 (스캔형 미지원) |
| Excel (.xlsx / .xls) | openpyxl | 시트별 셀 값, SQL 컬럼 분석 적용 |
| XML (.xml) | ElementTree | 요소 텍스트 + 속성값 |

### 압축 파일 (v2.0 신규)

| 형식 | 처리 방식 |
|---|---|
| `.zip` | 내부 로그/문서 파일 자동 추출 |
| `.tar.gz` / `.tgz` | 내부 파일 자동 추출 |
| `.gz` | 단일 gzip 파일 스트리밍 해제 |

## 10. 탐지 항목 설명

### 오남용 탐지 항목

| 항목 | 탐지 기준 | 위험도 |
|---|---|---|
| **개인정보 노출 (PII_EXPOSURE)** | SQL SELECT절 PII 컬럼 포함 또는 로그에서 직접 PII 패턴 탐지 | CRITICAL~MEDIUM |
| **단일 대량 노출 (PII_RECORD_EXPOSURE)** | 실효 노출량(건수×PII컬럼): ≥1,000 MEDIUM / ≥10,000 HIGH / ≥50,000 CRITICAL | CRITICAL~MEDIUM |
| **업무외시간 조회 (AFTER_HOURS)** | 08:00~19:00 외 시간에 PII 조회 5건 초과 | HIGH |

### 과다조회 탐지 항목

| 항목 | 탐지 기준 | 위험도 |
|---|---|---|
| **과다조회 (EXCESSIVE_ACCESS)** | 일별 ≥500 경고 / ≥2,000 위험, 시간당 ≥100 경고 / ≥500 위험 | CRITICAL~HIGH |
| **대량 내보내기 (BULK_EXPORT)** | LIMIT/TOP/ROWNUM ≥1,000건 또는 SELECT * (WHERE 없음), 하루 3건 초과 | HIGH |

### 탐지되는 개인정보 유형

| 유형 | 코드 | 심각도 | 탐지 예시 | 마스킹 출력 |
|---|---|---|---|---|
| 주민등록번호 | RRN | CRITICAL | `800101-1000008` | `80****-*******` |
| 계좌번호 | ACCOUNT_NO | CRITICAL | `계좌: 1234567890123` | `123****890` |
| 신용카드번호 | CREDIT_CARD | CRITICAL | `4111-1111-1111-1111` | `4111-****-****-1111` |
| 이름 (쿼리 내) | NAME_IN_QUERY | HIGH | `cust_name='홍길동'` | `홍**` |
| 전화번호 | PHONE | HIGH | `010-1234-5678` | `010-****-5678` |
| 생년월일 | BIRTHDATE | HIGH | `birth=1990-01-15` | `1990-**-**` |
| 여권번호 | PASSPORT | HIGH | `M12345678` | `M1*****78` |
| 이메일 | EMAIL | MEDIUM | `hong@example.com` | `ho***@example.com` |
| 주소 | ADDRESS | MEDIUM | `서울시 강남구 테헤란로 123` | 부분 마스킹 |
| 사원번호 | EMP_ID_IN_QUERY | MEDIUM | `emp_id=EMP003`, `사번: K-12345` | `em*****03` |
| **IP 주소** | IP_ADDRESS | LOW | `192.168.1.55`, `10.0.5.200` | `192.168.1.**` |

> **주민번호**: 날짜 유효성 + 체크섬 검증으로 오탐 최소화  
> **신용카드**: Luhn 알고리즘 검증  
> **IP 주소**: 루프백(127.x), 브로드캐스트(0.0.0.0 / 255.255.255.255), 링크로컬(169.254.x) 제외

### 사원번호 탐지 키워드 (선행 필수)

```
emp_id · employee_id · empno · 사원번호 · 사번 · 직원번호 · staff_id · staff_no
```

탐지 형태: 숫자(`12345`), 영문+숫자(`EMP003`), 영문접두어+숫자(`K-12345`, `A0012345`)

## 11. 한글 숫자 탐지

로그에 숫자가 한글로 기재된 경우도 자동으로 탐지합니다.

### 한글 숫자 매핑

| 한글 | 숫자 | 한글 | 숫자 |
|---|---|---|---|
| 영 / 공 | 0 | 육 | 6 |
| 일 | 1 | 칠 | 7 |
| 이 | 2 | 팔 | 8 |
| 삼 | 3 | 구 | 9 |
| 사 | 4 | | |
| 오 | 5 | | |

### 변환 예시

| 입력 (한글 숫자) | 변환 결과 | 탐지 결과 |
|---|---|---|
| `공일공-일이삼사-오육칠팔` | `010-1234-5678` | PHONE 탐지 |
| `공 일 공 일 이 삼 사 오 육 칠 팔` | `01012345678` | PHONE 탐지 (공백 제거) |
| `010-일이삼사-오육칠팔` | `010-1234-5678` | PHONE 탐지 (혼합 처리) |
| `사일일일-일일일일-일일일일-일일일일` | `4111-1111-1111-1111` | CREDIT_CARD 탐지 |

**변환 규칙:**
- 한글 숫자 3글자 이상 연속 시 변환 적용
- 중간 공백(스페이스, 탭, 다중 공백) → 제거
- 하이픈(-) → 그대로 유지
- 변환 후 주민번호 체크섬·카드 Luhn 검증 동일 적용

## 13. 자주 묻는 질문

**Q. 브라우저가 자동으로 열리지 않아요.**  
→ 콘솔에 표시된 URL을 브라우저 주소창에 직접 입력하세요: `http://localhost:5000`

**Q. 분석이 시작되지 않아요.**  
→ 로그 파일과 분석 기간이 모두 입력됐는지 확인하세요. 오남용/과다조회 중 하나 이상 체크 필요.

**Q. 결과에 사용자가 0명으로 나와요.**  
→ 분석 기간과 로그 파일의 날짜가 일치하는지 확인하세요. 로그 형식을 `자동감지` → 다른 형식으로 바꿔 보세요.

**Q. PDF를 올렸는데 탐지가 안 돼요.**  
→ 스캔된 이미지형 PDF는 텍스트 추출이 불가합니다. 텍스트 레이어가 있는 PDF만 지원합니다.

**Q. 압축 파일 안의 일부 파일이 분석되지 않아요.**  
→ `.log .txt .gz .csv .json .audit .docx .pdf .xlsx .xml` 이외 확장자는 자동으로 제외됩니다.

**Q. 이전 대비 비교(▲↑)가 표시되지 않아요.**  
→ 최소 2번 이상 분석을 실행해야 이력이 쌓입니다. 처음 실행 시에는 비교 데이터가 없습니다.

**Q. 포트 5000이 이미 사용 중이에요.**  
→ `pii_checker.exe --port 8080` 또는 `python web_app.py --port 8080` 으로 변경하세요.

**Q. 다른 PC에서 접속하고 싶어요.**  
→ 서버 PC의 IP를 확인 후 `http://[서버IP]:5000` 으로 접속하세요. 방화벽에서 해당 포트를 허용해야 합니다.

**Q. 분석 중 오류가 발생해요.**  
→ 콘솔 창의 오류 메시지를 확인하세요. 웹 UI에서는 진행 로그 영역 하단에 오류 내용이 표시됩니다.
