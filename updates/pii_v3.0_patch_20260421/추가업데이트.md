## 변경 내용

### 1. PPT/PPTX 파서 추가 (`doc_extractor.py`)

- `.pptx` 파일 지원 추가 — python-pptx 없이 표준 라이브러리(`zipfile` + XML)로 구현
- 슬라이드별 텍스트 추출 후 PII 탐지
- 구형 `.ppt` 바이너리는 미지원 → 안내 메시지 출력 (PowerPoint에서 .pptx로 재저장 필요)
- 웹 업로드 확장자에 `.pptx`, `.ppt` 추가 (`web_app.py`)

### 2. Excel 업로드 오류 수정 (`doc_extractor.py`)

- **증상:** xlsx 파일 업로드 시 "File is not a zip file" 오류
- **원인:** 구형 `.xls` 바이너리 포맷을 `.xlsx`로 저장한 경우 openpyxl이 거부
- **수정:** openpyxl 호출 전 `zipfile.is_zipfile()` 선검사 추가, 명확한 한국어 안내 메시지 출력

### 3. 주민번호(RRN) 미탐지 수정 (`config.py`, `pii_detector.py`)

- **증상:** txt 파일에 주민번호가 있어도 탐지되지 않음
- **원인 1:** 성별코드 정규식이 `[1-4]`만 허용 → 외국인(5-9) 주민번호 패턴 매칭 자체 실패
- **원인 2:** 키워드 없는 순수 데이터 파일에서 RRN 체크섬 실패 시 조용히 드롭
- **수정:**
  - 성별코드: `[1-4]` → `[1-9]` (내국인 1-4 + 외국인 5-9)
  - `validate_rrn(strict=False)` 모드 추가: 키워드 없는 raw 데이터 파일에서는 날짜+성별코드만 검증, 체크섬 생략
  - SQL/구조적 로그(키워드 있음)는 기존 strict 체크섬 유지

### 4. IP 주소 오탐 수정 (`config.py`)

- **증상:** IP 주소가 아닌 값(버전 문자열 등)이 IP로 탐지
- **원인:** 룩비하인드 `(?<!\d)` 만 적용 → `v1.2.3.4`, `ver3.11.0.1` 같은 버전 문자열 오탐
- **수정:**
  - 룩비하인드: `(?<!\d)` → `(?<![a-zA-Z0-9_.])` (영문자·숫자·점·언더스코어 직후 매칭 차단)
  - 룩어헤드: `(?!\d)` → `(?![.\d])` (5옥텟 패턴 `1.2.3.4.5`에서 `2.3.4.5` 오탐 방지)

| 테스트 케이스 | 수정 전 | 수정 후 |
|---------------|---------|---------|
| `v1.2.3.4` | 탐지 (오탐) | 미탐지 |
| `1.2.3.4.5` | `2.3.4.5` 탐지 (오탐) | 미탐지 |
| `192.168.1.1` | 탐지 | 탐지 |
| `ip=10.0.0.1` | 탐지 | 탐지 |
